和常见方案相比，适合谁？

下面从「数据归属、工作方式、能力边界」出发做简要对比，便于你判断 Quiet Editor 是否匹配自己的工作流。

vs Notion

相对 Notion 的优势

  • 本地优先、离线可用：正文与附件在本地仓库目录中，不依赖在线编辑器才能读写；弱网或离线时仍可完整编辑与检索。
  • 标准 Markdown 与 Git 版本历史：以文件夹 + .md 为主，配合整库 Git 同步；历史在 Git 里可追溯、可对比，而不是锁在专有云端格式中。
  • 无订阅与账户体系绑定：不将笔记访问权与云端账号强绑定；适合希望「数据在自己磁盘与自有远程」的团队与个人。
  • 更轻的原生性能路径：Rust + Tauri 原生壳，大仓库检索与图谱交互更偏向本地算力，减少「所有操作都绕一圈云端」的体感延迟。

Notion 在协作数据库、看板与多端富文本协同上仍很强；若你强依赖实时协同表格与评论流，可继续以 Notion 为主、用 Quiet Editor 管「版本化的文字资产」。

vs Obsidian

相对 Obsidian 的优势

  • 内置 Git 同步心智：以仓库为边界的一键拉取/推送、多远程与代理、按仓库排除远程等，与笔记目录同一套 UI，减少插件拼凑成本。
  • 一体化区块编辑体验：段落/标题/任务/表格/代码/Mermaid 等在统一区块模型与斜杠菜单下完成，表格与代码编辑路径更「应用原生」而非纯文本补丁感。
  • Rust 后端检索与索引：BM25 全文检索在本地索引上完成，适合笔记量大、希望在同一应用内闭环搜索与打开路径的用户。
  • 更克制的扩展面：不依赖社区插件生态即可覆盖 Git、图谱、导入导出等主干能力，升级与排错路径相对集中。

Obsidian 的插件生态与主题极为丰富；若你已深度定制 Obsidian 工作流，可把 Quiet Editor 视为「另一条偏原生与一体化」的备选线。

核心能力

面向「长文写作 + 结构化笔记 + 仓库级同步」设计，界面保持 Sage & Vellum 色系与低干扰布局。

view_agenda

区块式编辑器

十余种区块类型、斜杠命令、多选与拖拽、沉浸式模式与编辑锁定；代码块由 CodeMirror 6 驱动，Mermaid 在块内直接预览。

hub

引用与知识图谱

[[仓库/路径/文章]] 建立双向引用与反向索引；图谱视图支持平移、缩放与过滤，便于理解笔记网络。

search

BM25 全文检索

本地索引与相关性排序；全局搜索支持键盘导航；编辑器内查找快速定位。

sync

Git 整库同步

工作目录级版本控制；多远程、SSH/HTTPS、HTTP(S) 与 SOCKS5 代理；可按仓库排除远程；可选将部分设置与元数据随仓库走。

palette

主题与语言

Sage & Vellum 设计体系与多主题切换；中英双语界面；可选性能模式以降低阴影开销、换取弱设备流畅度。

import_export

导入与导出

支持常见办公格式转入 Markdown；导出 Markdown / PDF；图片等资源可整理到 .assets/ 目录侧写。
